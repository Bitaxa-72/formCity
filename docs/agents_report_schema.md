# Отчет по агентам

## Назначение

Отчет хранит агентские сделки, начисления, оплаты, остатки к оплате и помесячное распределение сумм.

Исходные файлы:

- `Отчет по Агентам_на 24.03.2026.xlsx`
- `Отчет по Агентам_по 31.03.2026 включительно.xlsx`
- `Отчет по Агентам_по 30.04.2026 включительно.xlsx`

## Таблицы

### `agents_report_sources`

Один файл-срез.

Поля:

- `project` - проект, сейчас `obvodny`
- `snapshot_month` - месяц среза
- `snapshot_date` - точная дата среза из имени файла
- `file_name` - имя исходного файла
- `file_hash` - hash файла для контроля импорта

### `agents_report_deals`

Одна строка = одна сделка/строка агента из отчета.

Основные поля:

- `agent_name` - агент
- `unit_number` - номер помещения
- `buyer_name` - ФИО покупателя
- `ddu_number` - номер ДДУ
- `contract_date` - дата договора
- `area_sqm` - площадь
- `commission_base_amount` - база для расчета агентского вознаграждения
- `commission_rate` - ставка вознаграждения
- `commission_amount` - сумма агентского вознаграждения
- `act_total_amount` - итого сумма по акту
- `paid_amount` - оплачено
- `remaining_amount` - остаток к оплате
- `act_info` - номер и дата акта
- `budget_month` - месяц бюджета
- `ddu_assignment_amount` - ДДУ + уступка
- `ddu_amount` - ДДУ
- `assignment_amount` - уступка
- `furniture_amount` - меблировка
- `note` - примечание

### `agents_report_monthly_values`

Одна строка = одно значение из правых помесячных блоков отчета.

Поля:

- `deal_source_row` - строка сделки в исходном файле
- `value_kind` - тип значения
- `period_kind` - тип периода
- `period_month` - месяц, если значение относится к конкретному месяцу
- `value` - сумма

## Типы помесячных значений

- `ddu_schedule` - распределение суммы ДДУ
- `assignment_schedule` - распределение суммы уступки

## Типы периода

- `past_periods_total` - сумма прошлых периодов
- `month` - значение конкретного месяца

## Единицы измерения

- `rub` - рубли
- `sqm` - квадратные метры
- `rub_per_sqm` - рубли за квадратный метр

## Безопасность

Отчет содержит чувствительные данные:

- ФИО покупателей
- номера ДДУ
- номера и даты актов
- имена ИП/агентов, если они являются персональными данными

Эти поля сохраняются в БД для расчетов, но не должны выводиться пользователю напрямую. Для строк сделок стоит `is_sensitive=true`, а чувствительные поля перечислены в `sensitive_fields`.

В будущей backend-логике можно отвечать агрегатами: количество сделок, сумма агентского вознаграждения, оплачено, остаток к оплате, суммы по бюджетным месяцам. Вывод персональных строк допустим только после отдельной маскировки.
