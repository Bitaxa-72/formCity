# Отчет об исполнении плана продаж

## Назначение

Отчет хранит показатели исполнения плана продаж по проекту `obvodny`.

Исходные файлы:

- `Отчет об исполнении плана продаж (Обв.118)_28.02.2026.xlsx`
- `Отчет об исполнении плана продаж (Обв.118)_31.03.2026.xlsx`
- `Отчет об исполнении плана продаж (Обв.118)_30.04.2026.xlsx`

## Таблицы

### `sales_plan_execution_sources`

Один файл-срез.

Поля:

- `project` - проект, сейчас `obvodny`
- `snapshot_month` - месяц среза
- `snapshot_date` - точная дата среза из заголовка отчета
- `file_name` - имя исходного файла
- `file_hash` - hash файла для контроля импорта

### `sales_plan_execution_facts`

Одна строка = одно числовое значение по блоку, сегменту, метрике, владельцу и сценарию.

Поля маршрутизации:

- `project` - проект
- `snapshot_month` - месяц среза отчета
- `block_kind` - блок отчета
- `segment` - сегмент объекта
- `metric_key` - нормализованная метрика
- `owner_scope` - общий показатель / застройщик / Велл
- `period_kind` - тип периода
- `period_month` - месяц, если блок месячный
- `year` - год, если блок годовой
- `scenario` - тип значения
- `value` - числовое значение
- `unit` - единица измерения

## Блоки

- `segment_cumulative` - верхний блок по сегментам: план, факт, отклонение, факт+прогноз, отклонение
- `month` - блок конкретного месяца: план, прогноз, факт, разница факт-прогноз
- `year` - блок `итого 2026`: план, прогноз, факт+актуализированный прогноз, факт, остаток к продаже
- `project_lifetime` - общий итог проекта: план, прогноз, факт+актуализированный прогноз, факт, остаток к продаже

## Сегменты

- `project_total` - итого проект
- `apartments` - апартаменты
- `restaurant` - ресторан

## Метрики

- `sales_revenue` - продажи
- `cash_receipts` - поступление денежных средств
- `contract_area_sqm` - объем законтрактованных площадей, м2
- `contract_count` - объем законтрактованных площадей, шт.
- `price_per_sqm` - цена за 1 м2

## Владелец

- `all` - общий показатель
- `developer` - застройщик
- `well` - Велл

## Сценарии

- `plan` - план
- `fact` - факт
- `deviation` - отклонение
- `forecast` - прогноз
- `fact_forecast` - факт+прогноз
- `forecast_deviation` - отклонение по факту+прогнозу
- `fact_minus_forecast` - разница факт-прогноз
- `fact_actualized_forecast` - факт+актуализированный прогноз
- `remaining_to_sell` - остаток к продаже

## Единицы измерения

- `rub` - рубли
- `sqm` - квадратные метры
- `count` - количество
- `rub_per_sqm` - рубли за квадратный метр

## Важные правила

Блоки нельзя смешивать без явного намерения пользователя. Например, `month` отвечает за конкретный месяц среза, а `project_lifetime` отвечает за общий итог проекта.

Если период не указан в будущей логике бота, разумный дефолт для этого отчета - последний доступный срез.

Для запросов по исполнению плана важны три независимых слоя:

- метрика: продажи, поступления, площадь, количество, цена за м2
- сценарий: план, факт, отклонение, прогноз, остаток к продаже
- блок: месяц, итого год, итого проект, верхний накопительный блок по сегментам
